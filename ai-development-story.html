AI × 设计师 = 无限可能

1缘起：为什么要用 AI 开发网站？

作为一个设计师，我一直想要一个简洁、现代的个人作品集网站来展示我的作品。但是，传统的开发流程对我来说总是充满挑战：需要学习 HTML/CSS/JS、搭建开发环境、处理响应式布局、调试兼容性……这些技术细节往往会让创意停滞。

直到我遇到了 AI 助手——它不仅能理解我的设计意图，还能立即将想法转化为代码。这就是"兰博的设计小摊"的诞生故事。

"AI 不是替代设计师，而是让设计师能够专注于创意，把技术实现交给 AI 伙伴。"

2传统开发 vs AI 辅助开发

让我们先看看两种开发方式的对比：

⏰ 传统开发流程

  • 学习 HTML/CSS/JS 语法（数周）
  • 搭建开发环境（数小时）
  • 手写每一个页面（数天）
  • 调试响应式布局（数小时）
  • 修复兼容性 bug（数小时）
  • 购买域名和服务器（$$$）
  • 部署和配置（数小时）

🚀 AI 辅助开发

  • 用自然语言描述需求（几分钟）
  • AI 立即生成代码（几秒钟）
  • 实时预览和调整（实时）
  • AI 自动处理响应式（自动）
  • AI 自动修复 bug（实时）
  • 使用免费 GitHub Pages（免费）
  • AI 帮你一键部署（几分钟）

3开发过程全记录

以下是我和 AI 助手协作开发"兰博的设计小摊"的完整时间线：

🎨 第一步：确定设计和结构

我告诉 AI："我想要一个简洁现代的个人作品集网站，包含首页、作品集、关于我、文章和联系我这几个页面。用纯 HTML/CSS/JS，不要框架。"

AI 立即理解了我的需求，为我创建了完整的文件结构：

// 网站结构
📁 项目根目录
├── index.html // 首页
├── portfolio.html // 作品集
├── about.html // 关于我
├── article.html // 文章列表
├── article-detail.html // 文章详情
├── work-detail.html // 作品详情
├── contact.html // 联系我
└── images/ // 图片资源

🛠️ 第二步：细节调整和优化

在开发过程中，我提出了一些具体的调整需求：

  • PC 端内容区域宽度调整：我要求将文章详情页的内容区域宽度改为与首页一致（1280px），AI 立即修改了 CSS 的 max-width 属性。
  • 缩小竖向间距：我觉得页面区块之间的间距太大，AI 帮我调整了 padding 和 margin 值，让页面更紧凑。
  • 修复返回按钮对齐：work-detail.html 的返回按钮没有和白色底块左对齐，AI 通过调整 padding 值修复了这个问题。

🔗 第三步：检查并修复跳转链接

我意识到网站的各个页面之间需要正确跳转。我让 AI "检查所有页面的跳转链接是否正常"。

AI 使用工具扫描了所有 HTML 文件，发现了 4 处错误的链接（指向 # 的空链接），并立即帮我修复：

// 修复前
<a href="#">首页</a>

// 修复后
<a href="index.html">首页</a>

🎨 第四步：替换 Logo

我准备好了一个 logo 图片（logo-white@1x.png），想替换掉页面底部占位用的文字 Logo。

我告诉 AI："底部的 logo 换上文件 logo-white，我已经放在文件夹里了，替换掉吧。"

AI 立即帮我替换了所有 7 个页面的 footer logo，从文字版本：

// 替换前
<div class="footer__logo">L</div>

// 替换后
<div class="footer__logo">
  <img src="images/logo-white@1x.png" alt="Logo">
</div>

🚀 第五步：部署到外网

最激动人心的时刻来了！我告诉 AI："这个网站帮我弄成外网能访问的。"

AI 帮我完成了以下所有步骤：

  1. 初始化 Git 仓库并提交代码
  2. 使用 GitHub API 创建远程仓库（Rambo-s-design-stall）
  3. 推送代码到 GitHub
  4. 启用 GitHub Pages
  5. 提供外网访问地址

整个过程只用了几分钟！现在任何人都可以访问我的网站了：https://rambolan.github.io/Rambo-s-design-stall/

5AI 开发的心得体会

通过这次与 AI 协作开发个人网站的经历，我总结了以下几点心得：

1. 用自然语言沟通
不需要懂代码语法，只需要用日常语言描述你的需求，AI 就能理解并执行。

2. 实时迭代和调整
看到不满意的地方，立即告诉 AI，它会马上帮你修改。这种实时反馈让创作过程更加流畅。

3. AI 不会犯错吗？
偶尔 AI 也会犯错（比如链接错误、对齐问题），但它能快速发现并修复。关键是要多检查、多测试。

4. 部署不再可怕
以前部署网站需要购买服务器、配置域名，现在 AI 可以帮你一键部署到 GitHub Pages，完全免费！

5. 保持创造力
AI 负责技术实现，你负责创意和方向。这种分工让你可以更专注于设计本身。

6你也可以做到！

如果你也想用 AI 开发自己的个人网站，这里有一些建议：

  • 明确你的需求：先想清楚网站要包含哪些页面、展示什么内容。
  • 选择合适的工具：像我一样使用 AI 助手（比如 WorkBuddy），它可以理解你的需求并生成代码。
  • 不要害怕尝试：AI 让试错成本变得很低，大胆提出你的想法，看到不满意就改。
  • 学习基础知识：虽然 AI 可以帮你写代码，但了解一些 HTML/CSS 的基础知识会让你和 AI 的沟通更高效。
